## Business Assumptions

- Every Return belongs to a valid Order Detail.
- A Return can only be created after the corresponding Order has been placed.
- Returned Quantity cannot exceed the quantity purchased in the corresponding Order Detail.
- Each Return record is associated with only one Order Detail.
- Most Returns involve a single unit, while multi-unit Returns are relatively uncommon.
- Return Status follows the business return workflow from **Requested** to **Approved**, **Rejected**, **Refunded**, or **Replaced**.
- Approved and Refunded Returns increase inventory only if the returned product is saleable.
